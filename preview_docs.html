Builder Kit Deployment Roadmap

This roadmap describes how the Builder Kit turns a prospective client intake into a branded Rally-Org environment, powered by Rally-IQ, and deployed into the client organization's own repository and landing zone.

The System Model

1. Builder KitClient-facing onboarding portal for intake, branding, structure, preview, and repo generation.
2. Rally-OrgThe operating framework that separates executive, operational, and coaching responsibilities.
3. Rally-IQThe intelligence layer for rosters, tournaments, communications, practice planning, player development, fundraising, and media.
4. Client Landing ZoneA branded repo and deployable site for the client organization.
Builder Kit → Rally-Org → Rally-IQ → Client Repo Landing Zone

Builder Kit Purpose

The Builder Kit is not just a deployment utility. It is the sales and onboarding experience that shows a potential client what their organization will look like after Rally-Org setup.

Client Intake

  • Organization name and region
  • Mission and operating priorities
  • Age groups and programs
  • Leadership roles
  • Current operational pain points

Brand Touch

  • Logo and visual identity
  • Primary and accent colors
  • Typography preferences
  • Voice and tone
  • Landing page preview

Rally-Org Operating Structure

Rally-Org creates the organizational structure that protects coaching quality by moving operational work away from coaches and into defined leadership and administrative roles. 

Executive Director
├── Operations Director
│   ├── Tournament Coordinator
│   ├── Communications Coordinator
│   ├── Fundraising Coordinator
│   └── Registration / Compliance Coordinator
├── Coaching Director
│   ├── Head Coaches
│   ├── Assistant Coaches
│   └── Player Development Staff
└── Finance / Sponsorship Director
    ├── Budget Management
    ├── Sponsor Relations
    └── Scholarship / Aid Tracking

The goal is simple: coaches coach, operators operate, directors direct, and families receive clear communication.

Team / Roster Page Documentation

The Builder Kit should demonstrate the public team roster page because it is one of the clearest ways for a client to understand how branding, consent, player identity, statistics, and media come together.

Documentation rule: use demo or consent-approved player data only. For client demos, the safe default is first name, last initial, jersey number, and placeholder cards until guardian consent is recorded.

Roster Page Requirements

  • Show active player cards with jersey number, positions, metrics, badges, and profile calls to action.
  • Show unused roster spots as placeholders so directors can see roster capacity and onboarding status.
  • Display counts such as profiles live and total roster spots.
  • Use the client's branding tokens from the Builder Kit for colors, type, cards, badges, and buttons.
  • Preserve privacy defaults until the organization has signed media releases.

Individual Player Profile Documentation

Individual player profiles should be documented as the bridge between Rally-Org's public trust model and Rally-IQ's private player development data. A profile can include statistics, photos, clips, badges, recruiting context, and player development history once consent and publishing rules are satisfied.

Profile Data

  • Jersey number and positions
  • Core stats and player badges
  • Coach-approved player summary
  • Development notes approved for public view

Publishing Controls

  • Guardian media release required
  • Full profile can be disabled per player
  • Photos and clips are opt-in
  • Rally-IQ remains the private source of truth

Player Video Wall and GameChanger Clip Flow

Player profiles should support a media wall where clips exported from GameChanger or another film source can be organized by result, date, inning, event type, or coach-created category.

Media Flow

GameChanger / film source
  → exported clip metadata
  → Rally-IQ media record
  → player profile association
  → consent and publish check
  → branded public video wall
  → embedded clip playback

Onboarding Flow for Client Demo

  1. Enter organization basics and mission.
  2. Choose brand colors, logo, typography, and tone.
  3. Define Executive Director, Operations, Coaching, Finance, and Communications roles.
  4. Map coaches away from operational tasks.
  5. Preview the branded Rally-Org landing page.
  6. Preview the team roster page with safe placeholder cards.
  7. Preview an individual player profile and video wall using approved demo assets.
  8. Generate org configuration and theme tokens.
  9. Create or populate the client repo landing zone.
  10. Deploy the first branded page to GitHub Pages, Vercel, or another host.

Deployment Options

GitHub Pages

Best for the first branded landing-zone demo.

git add index.html org.config.json theme.tokens.json .github/workflows/pages.yml
git commit -m "Create Rally-Org client landing zone"
git push

GitHub Pages workflow

name: Deploy Client Landing Zone

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    environment:
      name: github-pages
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: .
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4

Roadmap

Phase 1 — Demo Builder

  • Static intake form
  • Brand preview
  • Generated org structure
  • Generated roster preview
  • Generated player profile preview
  • Downloadable HTML landing page

Phase 2 — Repo Generator

  • Create client repo from template
  • Commit generated config and branding
  • Add GitHub Pages workflow
  • Create client README and onboarding checklist

Phase 3 — Rally-IQ Integration

  • Connect roster, tournament, communications, fundraising, stats, and media modules
  • Add role-based access for directors, coaches, coordinators, and families
  • Add dashboard views for executive and coaching separation

Phase 4 — Client Launch Package

  • Deploy public landing zone
  • Invite client leadership
  • Configure first programs and teams
  • Transition from demo to operating environment

Positioning Statement

Rally-Org Builder Kit takes an organization from intake to identity to structure to deployment. It creates a branded client landing zone and establishes the operating foundation that Rally-IQ powers after launch. 

Rally-Org creates the structure.
Rally-IQ powers the execution.
Builder Kit launches the client environment.
